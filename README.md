# Network Monitor

Простое **GUI‑приложение** на Python/Tkinter для мониторинга сетевой инфраструктуры: проверка доступности хостов (ping), контроль TCP‑портов, приоритеты узлов, история проверок и экспорт отчётов в HTML/TXT.

## Возможности

- Мониторинг узлов по IP или hostname.
- Проверка доступности по ICMP (ping) с измерением задержки.
- Проверка TCP‑портов (например, 80, 443, 3389 и др.).
- Классификация узлов по категориям (Серверы, Сетевое оборудование, Принтеры и т.п.).
- Приоритеты узлов: критичный, важный, обычный (с иконками 🔴/🟡/🟢 в таблице).
- История проверок (до 1000 записей) с отдельным окном просмотра.
- Автообновление (30 сек, 1, 2, 5, 10 минут или выкл.).
- Фильтр по категориям и кнопка показа только offline‑узлов (`🔴 OFFline`).
- Звуковое и визуальное предупреждение при падении критичных узлов.
- Экспорт отчёта о состоянии узлов в HTML и TXT.

## Требования

- Windows (используется `ping -n`, `winsound`, `CREATE_NO_WINDOW`).
- Python 3.8+ (желательно свежая версия 3.x).
- Модуль `tkinter` (обычно входит в стандартную поставку Python).
- Права на выполнение команды `ping` в системе.

Сторонние пакеты не используются, только стандартная библиотека Python.

## Установка и запуск

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/your-username/network-monitor.git
   cd network-monitor
   ```

2. (Необязательно) Создайте виртуальное окружение:
   ```bash
   python -m venv venv
   venv\Scripts\activate
   ```

3. Установите зависимости (фактически пусто, но на всякий случай):
   ```bash
   pip install -r requirements.txt
   ```

4. Запустите приложение:
   ```bash
   python network_monitor.py
   ```

## Хранение данных

Приложение использует два места для данных:

- Конфигурация: файл `config.json` в `%APPDATA%\NetworkMonitor`, где хранится путь к рабочей папке данных.
- Рабочая папка данных (выбирается при первом запуске или через меню «Файл → Выбрать папку данных…»), в ней:
  - `nodes.json` — список узлов.
  - `categories.json` — список категорий.
  - `history.json` — история проверок (до 1000 последних записей).

Данные пользователя (узлы, история) не должны попадать в git‑репозиторий, их можно исключить через `.gitignore`.

## Работа с узлами

### Добавление узла

Через меню «Узлы → Добавить узел…» или кнопку «➕ Добавить узел» в верхней панели.

В диалоге можно задать:
- Название.
- IP/hostname.
- Категорию (можно выбрать существующую или ввести новую).
- Приоритет (критичный / важный / обычный).
- Список портов через запятую, например: `80, 443, 3389`.
- Комментарий.

### Редактирование и удаление

- Редактирование: выбрать узел в таблице и нажать «✏️ Редактировать» (или двойной клик по строке).
- Удаление: выбрать узел и нажать «🗑️ Удалить» (с подтверждением).

### Категории

Отдельное окно «Управление категориями…» позволяет:
- Добавлять новые категории.
- Удалять существующие (с подтверждением).

Фильтрация по категориям доступна через выпадающий список «Категория» в панели инструментов.

## Проверка состояния

### Проверка всех узлов

Кнопка «🔄 Проверить все» или меню «Проверка → Проверить все»:
- Для всех узлов запускается параллельная проверка (ping + порты).
- В таблице отображаются:
  - Статус: ✓ Online / ✗ Offline / ⚠ Высокий пинг / ⏳ Проверка….
  - Пинг в мс.
  - Результат проверки каждого порта (✓порт / ✗порт / номер без статуса).

### Проверка одного узла

Через меню «Проверка → Проверить выбранный» или по выделенному узлу:
- Проверяется только один узел, результат обновляется в таблице и истории.

### Фильтр по offline

Кнопка `🔴 OFFline`:
- При нажатии включает режим отображения только тех узлов, у которых статус `offline`.
- Кнопка меняет вид (красный фон, белый текст, галочка) при активном фильтре.

При этом внизу отображается статистика:
- Общее число узлов, количество online/offline, а также количество offline, показанных с учётом фильтра.

### Автообновление

Комбо «Автообновление»:
- Значения: Выкл, 30 сек, 1/2/5/10 мин.
- При включении периодически вызывается «Проверить все», пока автообновление не отключено.

## История проверок

Меню «Проверка → История проверок» открывает отдельное окно с таблицей:
- Время.
- Название узла.
- IP.
- Статус (с текстовыми значками).
- Задержка (ms).

Хранится до 1000 последних записей; новые записи добавляются при каждой проверке.

## Оповещения о критичных узлах

Если при проверке всех узлов какие‑то узлы с приоритетом `Критичный` оказываются `offline`, приложение:
- Воспроизводит системный предупреждающий звук (`winsound.MessageBeep`).
- Показывает окно с перечнем недоступных критичных узлов.

Это помогает быстро заметить падение важных сервисов.

## Экспорт узлов и отчётов

### Импорт/экспорт списка узлов

Меню «Файл → Импорт узлов… / Экспорт узлов…»:
- Формат: JSON, структура совместима с внутренним `nodes.json`.
- При импорте узлам назначаются новые ID, чтобы не конфликтовать с уже существующими.

### Экспорт отчёта

Меню «Файл → Экспорт отчёта (HTML/TXT)…»:
- HTML‑отчёт: страница со сводкой (сколько узлов, сколько online/offline) и таблицей по каждому узлу.
- TXT‑отчёт: текстовый файл с блоком по каждому узлу и краткой сводкой.

## Иконка приложения

При наличии файла `icon.ico` в каталоге с `network_monitor.py` приложение использует его как иконку окна.  
Если файла нет, приложение просто продолжит работу без ошибки.
