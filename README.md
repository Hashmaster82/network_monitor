# Network Monitor

Простое **GUI‑приложение** на Python/Tkinter для мониторинга сетевой инфраструктуры: проверка доступности хостов (ping), контроль TCP‑портов, приоритеты узлов, история проверок и экспорт отчётов в HTML/TXT.[file:1]

## Возможности

- Мониторинг узлов по IP или hostname.
- Проверка доступности по ICMP (ping) с измерением задержки.[file:1]
- Проверка TCP‑портов (например, 80, 443, 3389 и др.).[file:1]
- Классификация узлов по категориям (Серверы, Сетевое оборудование, Принтеры и т.п.).[file:1]
- Приоритеты узлов: критичный, важный, обычный (с иконками 🔴/🟡/🟢 в таблице).[file:1]
- История проверок (до 1000 записей) с отдельным окном просмотра.[file:1]
- Автообновление (30 сек, 1, 2, 5, 10 минут или выкл.).[file:1]
- Фильтр по категориям и кнопка показа только offline‑узлов (`🔴 OFFline`).[file:1]
- Звуковое и визуальное предупреждение при падении критичных узлов.[file:1]
- Экспорт отчёта о состоянии узлов в HTML и TXT.[file:1]

## Требования

- Windows (используется `ping -n`, `winsound`, `CREATE_NO_WINDOW`).[file:1]
- Python 3.8+ (желательно свежая версия 3.x).
- Модуль `tkinter` (обычно входит в стандартную поставку Python).
- Права на выполнение команды `ping` в системе.[file:1]

Сторонние пакеты не используются, только стандартная библиотека Python.[file:1]

## Установка и запуск

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/your-username/network-monitor.git
   cd network-monitor
   ```

2. (Необязательно) Создайте виртуальное окружение:
   ```bash
   python -m venv venv
   venv\Scripts\activate
   ```

3. Установите зависимости (фактически пусто, но на всякий случай):
   ```bash
   pip install -r requirements.txt
   ```

4. Запустите приложение:
   ```bash
   python network_monitor.py
   ```

## Хранение данных

Приложение использует два места для данных:

- Конфигурация: файл `config.json` в `%APPDATA%\NetworkMonitor`, где хранится путь к рабочей папке данных.[file:1]
- Рабочая папка данных (выбирается при первом запуске или через меню «Файл → Выбрать папку данных…»), в ней:
  - `nodes.json` — список узлов.
  - `categories.json` — список категорий.
  - `history.json` — история проверок (до 1000 последних записей).[file:1]

Данные пользователя (узлы, история) не должны попадать в git‑репозиторий, их можно исключить через `.gitignore`.[file:1]

## Работа с узлами

### Добавление узла

Через меню «Узлы → Добавить узел…» или кнопку «➕ Добавить узел» в верхней панели.[file:1]

В диалоге можно задать:
- Название.
- IP/hostname.
- Категорию (можно выбрать существующую или ввести новую).[file:1]
- Приоритет (критичный / важный / обычный).[file:1]
- Список портов через запятую, например: `80, 443, 3389`.[file:1]
- Комментарий.

### Редактирование и удаление

- Редактирование: выбрать узел в таблице и нажать «✏️ Редактировать» (или двойной клик по строке).[file:1]
- Удаление: выбрать узел и нажать «🗑️ Удалить» (с подтверждением).[file:1]

### Категории

Отдельное окно «Управление категориями…» позволяет:
- Добавлять новые категории.
- Удалять существующие (с подтверждением).[file:1]

Фильтрация по категориям доступна через выпадающий список «Категория» в панели инструментов.[file:1]

## Проверка состояния

### Проверка всех узлов

Кнопка «🔄 Проверить все» или меню «Проверка → Проверить все»:
- Для всех узлов запускается параллельная проверка (ping + порты).[file:1]
- В таблице отображаются:
  - Статус: ✓ Online / ✗ Offline / ⚠ Высокий пинг / ⏳ Проверка….[file:1]
  - Пинг в мс.
  - Результат проверки каждого порта (✓порт / ✗порт / номер без статуса).[file:1]

### Проверка одного узла

Через меню «Проверка → Проверить выбранный» или по выделенному узлу:
- Проверяется только один узел, результат обновляется в таблице и истории.[file:1]

### Фильтр по offline

Кнопка `🔴 OFFline`:
- При нажатии включает режим отображения только тех узлов, у которых статус `offline`.[file:1]
- Кнопка меняет вид (красный фон, белый текст, галочка) при активном фильтре.[file:1]

При этом внизу отображается статистика:
- Общее число узлов, количество online/offline, а также количество offline, показанных с учётом фильтра.[file:1]

### Автообновление

Комбо «Автообновление»:
- Значения: Выкл, 30 сек, 1/2/5/10 мин.[file:1]
- При включении периодически вызывается «Проверить все», пока автообновление не отключено.[file:1]

## История проверок

Меню «Проверка → История проверок» открывает отдельное окно с таблицей:
- Время.
- Название узла.
- IP.
- Статус (с текстовыми значками).
- Задержка (ms).[file:1]

Хранится до 1000 последних записей; новые записи добавляются при каждой проверке.[file:1]

## Оповещения о критичных узлах

Если при проверке всех узлов какие‑то узлы с приоритетом `Критичный` оказываются `offline`, приложение:
- Воспроизводит системный предупреждающий звук (`winsound.MessageBeep`).[file:1]
- Показывает окно с перечнем недоступных критичных узлов.[file:1]

Это помогает быстро заметить падение важных сервисов.

## Экспорт узлов и отчётов

### Импорт/экспорт списка узлов

Меню «Файл → Импорт узлов… / Экспорт узлов…»:
- Формат: JSON, структура совместима с внутренним `nodes.json`.[file:1]
- При импорте узлам назначаются новые ID, чтобы не конфликтовать с уже существующими.[file:1]

### Экспорт отчёта

Меню «Файл → Экспорт отчёта (HTML/TXT)…»:
- HTML‑отчёт: страница со сводкой (сколько узлов, сколько online/offline) и таблицей по каждому узлу.[file:1]
- TXT‑отчёт: текстовый файл с блоком по каждому узлу и краткой сводкой.[file:1]

## Иконка приложения

При наличии файла `icon.ico` в каталоге с `network_monitor.py` приложение использует его как иконку окна.[file:1]  
Если файла нет, приложение просто продолжит работу без ошибки.
